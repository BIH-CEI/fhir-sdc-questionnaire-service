FHIR SDC Form Manager

A containerized FHIR Form Manager — HAPI FHIR with the SDC IG and MII PRO IG baked in — plus the validation infrastructure that proves it conforms.

What this is

Two things, both shipped as code:

  1. A container image — HAPI FHIR with SDC 3.0.0 and the MII PRO IG already loaded at build time. No first-boot IG download, no terminology bootstrap, no flaky cold start.
  2. Validation infrastructure — CI smoke tests, SDC compliance tests, integration tests, FHIR Validator integration. Every published image has been booted and asserted against the SDC Form Manager capability statement before the tag was pushed.

This is not a workshop or a demo deployment. Those experiments lived here previously and have been removed.

Run it

Pull and start

docker pull ghcr.io/BIH-CEI/fhir-sdc-form-manager:latest
docker run -p 8080:8080 ghcr.io/BIH-CEI/fhir-sdc-form-manager:latest

# FHIR endpoint:
curl http://localhost:8080/fhir/metadata

First-boot reachable in ~60–90s on amd64. No external dependencies — Postgres is optional and only needed for persistence.

Validation infrastructure

Living in api/tests/ and .github/workflows/build-and-publish.yml:

  • CI smoke test — every PR boots the freshly built image, hits /fhir/metadata, and asserts at least 20 MII PRO StructureDefinitions are loaded. If the image won't boot or the IG didn't install, the build fails.
  • SDC compliance tests (api/tests/sdc_compliance/) — assertions against the SDC Form Manager CapabilityStatement: metadata, OperationOutcome shapes, $package.
  • Integration tests (api/tests/integration/) — Questionnaire CRUD, calculated-expression $extract, PRO-CTCAE scoring round-trip.
  • FHIR Validator integration — official HL7 validator alongside the pytest suite (see FHIR_VALIDATOR_INTEGRATION.md).
  • Coverage map — TEST_REQUIREMENTS_MAPPING.md tracks which SDC Form Manager requirements are covered by which tests.

SDC capabilities

Implements the SDC Form Manager CapabilityStatement:

OperationStatusImplementation
$populateSupportedHAPI Clinical Reasoning
$extractSupportedHAPI Clinical Reasoning
$packageSupportedFastAPI sidecar (HAPI native is partial)
$localizeSupportedFastAPI sidecar
$assembleNot implementedOptional in spec — see ASSEMBLE_OPERATION_SUMMARY.md

SHALL resources — Questionnaire, ValueSet (+ $expand, $validate-code), CodeSystem (+ $lookup), Library — all CRUD via HAPI.
